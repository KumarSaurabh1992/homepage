Asynchronus Finite Difference for Advection-Diffusion Equation

Conference Paper : Kumar Saurabh, "Implementation of Asynchronous Scheme on Parallel Frameworks using MPI." in 18th AeSI Annual CFD Symposium. Link

This work was a part of Master Thesis. The thesis and presentations can be found here

The numerical solution of partial differential equations in large-scale problems requires the usage of parallel computing. In this context, the computational domain is often split into smaller sub-domains that are handled by separate processors. Due to the non-local nature of the underlying problem, communication as well as synchronization between these processing elements (PEs) is required. Finite Difference is one of such techniques to solve the PDEs. Since the discretization by finite- difference schemes results in mostly simple arithmetic operations, the overhead due to communication and synchronization might become a bottleneck. Current trends in massively parallel computing systems suggest that the number of processors will continue to grow over time, which in turn, increases the overhead associated with communication and synchronization. Simulation at extreme levels of parallelism will then require an elimination, or at least a tight control of these overheads. While stability is conserved when these schemes are used asynchronously, accuracy is greatly degraded. Message arrivals at PEs are essentially random processes, so is the behavior of the error. Within a statistical framework the average errors drop always to first-order regardless of the statistics of delay.

The Asynchrony Tolerant (AT) scheme proposed by Donzis & Aditya 1 and Mudigree et. al.2 are capable of maintaining the order of accuracy even in presence of delays and robust the asynchrony. A rigorous analysis of these scheme is performed. However, in all of these scheme the error increases with the increase in the delay. The two MPI communication technique, namely MPI Send/Recv and MPI Put/Get is compared for the statistics of delay. Further, the effect of distributing the work across various nodes is investigated.

Key Findings

  • MPI_Put/Get outperforms MPI_Send/Recv in delay statistics. This can be explained due to the fact that there is buffering of data in MPI_SendRecv while in the case of MPI_Put/Get one processor writes directly in the window created by the another processor. Thus for the Asynchronous operation, MPI_Put/Get is preferred.
  • The communication among the processor within the same node requires less time than communicating across the nodes. Thus, it should be expected that the distribution of processor across the node would take more time. But for a job requiring large Cache Size, this is not the case. This can be explained by the fact that distributing the processor across the nodes gives access to more RAM, which in turn reduces the number of Cache Miss and Hit.

References

  • Mudigree, D., Sherleker,S., Ansumali, S., 2014 “Delayed Difference scheme for large scale scientific simulations”, Physical Review Letters, 113(21), 218701.

Prediction of API Gravity of Crude Oils using SARA Analysis: Computational Intelligence based Models

Publication: Purva Goel, Kumar Saurabh, Veena Patil-Shinde and Sanjeev Tambe. "Prediction of API Gravity of Crude Oils using SARA Analysis: Computational Intelligence based Models". Accepted in Society of Petroleum Engineering. Paper Id: SPE-184391-PA

The API gravity (API) is an important physio-chemical characteristic of crude oils and often used in determining their properties and quality. There exist models — predominantly linear ones — for predicting the API magnitude from the molecular composition of a crude oil. This approach is tedious and time-consuming since it requires quantitative determination of numerous crude oil components. Usually, the hydrocarbons present in the crude oils are grouped according to their molecular average structures into Saturates, Aromatics, Resins and Asphaltenes (SARA) fractions. The non-linear relationship between SARA fraction and API gravity is modeled using three Computer Intelligence based models namely Genetic Programming, Artificial Neural Network and Support Vector Regression. In order to verify the true nature of the dependencies (whether linear or nonlinear) between the SARA constituents and the corresponding API gravity values, a large data-set consisting of SARA analyses of 565 crude oil samples was compiled from a number of research articles including a database. The Steiger's Z-test (Steiger, 1980) was performed for rigorously comparing the prediction and generalization performance of the three CI-based model.

Key Findings

  • It has been also found that each of the three CI-based nonlinear models possesses a better API gravity prediction accuracy and generalization capability than not only the original improved linear version
  • The The best fitting GP-based API gravity model developed in this study possesses a nonlinear form giving the evidence of nonlinear relationship.
  • A comparison of the prediction and generalization performance of the three CI-based models indicates that the SVR strategy has yielded an overall best performing model.
